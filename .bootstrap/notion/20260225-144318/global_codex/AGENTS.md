# ~/.codex/AGENTS.md (Global)

> 모든 프로젝트에서 공통으로 적용되는 “전역 작업 합의(Working Agreements)”.
> 프로젝트별 규칙(폴더 구조/스택/문서 규칙)은 레포 루트 `AGENTS.md`를 우선합니다.

## 1) 커뮤니케이션

- **모든 답변은 한국어**로 작성합니다.
  - 코드/명령어/라이브러리명/에러 메시지는 원문을 유지하되, 설명은 한국어로 합니다.
- 불확실한 내용은 “확실/추정”을 구분해서 말합니다.
- 과도한 장황함보다 “지금 당장 실행 가능한 지침/결과”를 우선합니다.

## 2) 문서 작성 언어/인코딩

- 문서(마크다운/텍스트)는 **한국어**로 작성합니다.
- 새로 만드는 텍스트 파일은 **UTF-8(가능하면 BOM 없음)** 으로 저장합니다.
- 기존 파일이 다른 인코딩이면 **임의로 인코딩을 변경하지 말고** 원본을 유지합니다.

## 3) 파일/폴더 삭제 및 이동 금지

- **삭제는 절대 하지 않습니다.**
  - `rm`, `del`, `git rm` 등 직접/간접 삭제 동작 포함
  - “정리”가 필요하면 삭제 대신: `old/` 보관, `.bak` 백업, DEPRECATED 표시 등을 사용합니다.
- **이동/리네임은 반드시 먼저 물어보고 진행합니다.**
  - `mv`, `ren`, `git mv` 등 경로 변경/리네임 포함
  - 물어볼 때: “왜 필요한지 / 영향 범위 / 롤백 방법”을 함께 제시합니다.

## 4) 민감정보/비밀정보

- 토큰/키/개인정보/쿠키/계정정보를 코드/문서/로그에 절대 기록하지 않습니다.
- 예시는 `REDACTED`, `YOUR_TOKEN_HERE` 같은 플레이스홀더로 대체합니다.

## 5) 작업 방식(최소 규율)

- 큰 변경은 **Plan → Execute** 순서로 진행합니다.
- 파일 이동/리네임이 필요해 보이면, 실행 전에 반드시 질문합니다.

## 6) Antigravity 협업 위임 정책 (전역 기본)
- Supabase CLI 실행, E2E/UI 테스트, 시각 검증, 이미지/에셋 생성·가공은 기본적으로 Google Antigravity에 위임합니다.
- Codex는 코드 변경/리팩터링, 문서화, 단위 테스트/타입체크/린트/코드 로직 검증을 담당합니다.
- 프로젝트 루트 `AGENTS.md`에 별도 정책이 있으면 프로젝트 정책을 우선합니다.
- Playwright/Playwright MCP는 프로젝트 정책이 명시적으로 허용한 경우에만 사용합니다.
